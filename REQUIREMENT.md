# 网页拼图小游戏（Nuxt 4 + Vercel 一键部署）需求

## 一、项目整体说明
网页休闲拼图平台，包含 **休闲、云冒险、自拍上传** 三大游玩模式。内置：图片远程数据库读取、不规则拼图切割、计分存档、排行榜、道具系统、广告复活、云端图片上传全套功能。游戏节奏宽松无时间紧迫感，代码分层清晰，附完整 README.md。

## 二、图库数据规则
1. 不依赖本地静态图片，全部从远程数据库接口拉取；封装独立图片请求组件/服务，数据逻辑和页面业务代码完全解耦。
2. 数据库图片按数字编号排序，组件严格按编号顺序加载，加载顺序固定不变。
3. 后台直接改数据库即可完成图片增删改，前端无需改代码/重新部署即可实时更新图库。
4. **接口暂无真实后端，全部封装为空函数/占位符（Promise 返回 mock 数据），在代码里注释清楚"后续对接后端修改此处"。**

## 三、游戏开局流程
用户进入首页仅选择三大游戏模式，无需额外选择图片分类；休闲模式自动按图库编号顺序加载图片开局。

## 四、三大游戏模式完整规则

### 1. 休闲模式
- 拼图切块 30~80 块，全部随机不规则异形切割，不使用均等网格分块；
- 素材全部读取远程数据库图库图片；
- 对局不计分，不参与排行榜。

### 2. 云冒险模式（唯一计分 & 排行榜模式）
- 无固定切块范围，加载其他用户"自拍上传的公开拼图"，完全沿用原图 + 玩家自定义切块难度，按上传时间顺序依次闯关；
- 模式入口增加说明弹窗，点击展示文案：「云冒险模式：按时间顺序游玩其他玩家自拍模式上传的公开拼图，完全沿用玩家上传时的图片、自定义切块难度进行闯关」；
- 单局得分 = 当前拼图总块数，总分本地持久化缓存 localStorage；
- 首页展示云冒险历史最高分，单独搭建排行榜页面，先用模拟假数据渲染，预留接口位置后续对接真实数据库；
- 每日游玩次数限制：自然日重置，每日免费 5 局；耗尽后提供两种解锁方式：观看激励广告兑换额外局数、付费开通永久无次数+免广告特权；
- 每日对局数量本地持久化存储，预留云端统计对接字段；
- 对局得分计入排行榜榜单。

### 3. 自拍上传模式
- 两种选图渠道：网页摄像头拍照（单次单张）、本地相册多选图片；
- 页面底部难度调节滑块，切块可调区间 30~200 块，默认随机 30~80；
- 确认难度后开始游戏，自定义图片同样使用随机不规则异形切割；
- 多图逻辑：按选中顺序依次开局，通关一张自动加载下一张，全部完成则结束流程；
- 单张拼图通关后弹窗询问"是否推荐本局"，选"推荐"展示云端上传按钮，可单独上传当前图片到公共图库供云冒险模式使用；仅完成前端 UI + 顺序游玩逻辑，预留上传接口；
- 本模式所有对局不计分、不上排行榜。

## 五、切割与倒计时规则
1. 全局统一不规则异形拼图切割，碎片尺寸、形状随机，摒弃规整均分网格；
2. 倒计时宽松：块数量越多，通关总时长同步加长，无紧张限时压力。

## 六、全局道具系统
1. **智能还原道具**：一键自动摆正 3 块错位拼图碎片；
2. **时间冻结道具**：暂停游戏倒计时 60 秒，冻结期间可正常拖拽拼图。

## 七、闯关失败 & 广告复活逻辑
1. 倒计时耗尽 → 判定失败，锁定所有游戏操作，弹出失败弹窗；
2. 支持观看激励广告原地复活本局（广告为占位 mock 弹窗）；
3. 广告完成后倒计时全额重置，继续当前拼图，无需重新选择模式。

## 八、项目技术硬性要求
1. 框架固定 **Nuxt 4**，原生适配 Vercel，无需额外配置，上传仓库即可一键部署（不写 `vercel.json` 也能跑；如需，也可提供最简配置）；
2. 目录分层规范：**pages / components / composables (工具函数&hooks) / services (接口请求) / stores (全局状态 pinia) / utils / assets** 完全分离，关键代码加注释；
3. 输出完整 **README.md**：项目目录结构、各页面功能、组件用途、部署步骤、后续对接数据库/后端修改位置；
4. 全站共用统一 **AppHeader 导航栏** + **AppFooter 底部版权**，样式极简通用，方便后续改配色文字；
5. 本地持久化：分数、每日次数、道具数量都存 localStorage；数据库/图片上传/排行榜接口全部为空函数占位，标注清楚后续对接后端位置。
